# HackTrader v0.7.7

Release date: 2026-04-22/23
Status: Active

## Summary
v0.7.7 is the stability release for HackTrader's market-data layer. It fixes the broken request contract between the dashboard, API endpoint, and breakout runner; promotes the Python runner as the single production path; improves degraded-mode visibility; adds health tracking; and introduces repeatable smoke tests.

## What changed

### Market-data contract fixes
- Fixed the `api.php -> run-brk.sh` argument order.
- Added explicit period normalization in `api.php`:
  - `1m -> 1min`
  - `5m -> 5min`
  - `1h -> 1h`
  - `1d -> 1day`
- Ensured `run-brk.sh` executes `run-brk.py` via an absolute script-dir path.

### Production runner
- `run-brk.py` is now the sole production breakout engine.
- `run-brk.c` remains on disk only as a legacy/experimental artifact.
- README now documents the Python production path explicitly.

### Live/stale observability
- Added `live_status` and `live_error_summary` to API responses.
- Dashboard now surfaces degraded stale-cache mode instead of hiding it.
- Added a dedicated live status chip in the UI.

### Usage tracking
- Moved request accounting authority into `api.php`.
- `api_usage_tracker.json` now records request count, provider outcomes, stale serves, and recent events.
- Removed duplicate Python-side request counting.

### Correlation endpoint
- Fixed malformed JSON content type in `correlate.php`.
- Added deterministic direct/sector/baseline fallback logic for known and unknown tickers.

### Verification and health checks
- Added `scripts/check-api-contract.py`.
- Added `scripts/smoke-market-data.sh`.
- Added `healthz.php`.
- Added `/var/www/html/state/health-status.json` tracking for:
  - total requests
  - live successes
  - cache hits
  - stale serves
  - errors
  - consecutive failures
  - consecutive stale serves
  - recent event ratios
